Assessment JSON Template Documentation

This guide explains the JSON format used for creating assessment templates in the ScalePad assessment system. Use this structure to create new assessments or modify existing ones.

üìã Overview

Assessment templates are JSON files that define the structure, content, and scoring logic for interactive assessments. Each assessment consists of metadata and a collection of categories, each containing multiple questions with criteria-based responses.

üìä Field Reference

This section details all available fields in the assessment template structure:

Root Level

Field Type Required Description
assessment_template_create_payload Object Required The main container for all assessment data

Assessment Template Payload

Field Type Required Description
title String Required The name of the assessment (e.g., "AI Readiness Assessment")
description String Required A brief description of what the assessment evaluates
is_category_weight_evenly_distributed Boolean Required Whether all categories should have equal weight in scoring
categories Array Required Array of category objects that group related questions

Category Object

Field Type Required Description
title String Required The name of the category (e.g., "Technology Foundation")
description String Required Description of what this category evaluates
is_question_weight_evenly_distributed Boolean Required Whether all questions in this category should have equal weight
questions Array Required Array of question objects

Question Object

Field Type Required Description
title String Required The question text (e.g., "System documentation quality")
description String Required Detailed explanation of what the question evaluates
remediation_tips String Optional Guidance for improving the assessed area
criterion_label_type_enum String Required Type of response criteria ("MultiResponse" is standard)
scoring_instructions String Required Guidance for how to score this question
criteria Array Required Array of possible response options

Criteria Object

Field Type Required Description
label_enum String Required Response level: "AtRisk", "NeedsAttention", "Satisfactory", or "NotApplicable"
description String Required Human-readable description of this response level

üéØ Response Levels

The assessment system uses four standard response levels to evaluate client readiness:

AtRisk

Indicates significant gaps or deficiencies that require immediate attention. This represents the lowest level of readiness and typically requires urgent remediation.

NeedsAttention

Shows moderate gaps that should be addressed in the near term. While not critical, these areas need improvement to reach optimal readiness.

Satisfactory

Indicates good practices are in place. This represents adequate readiness and may only need minor improvements or maintenance.

NotApplicable

Used when a question doesn't apply to the client's situation or business model. This option prevents irrelevant questions from affecting the overall score.

üí° Best Practices

Question Design

  • Be specific: Questions should evaluate concrete, measurable aspects of the client's environment
  • Use business language: Frame questions in terms of business impact, not just technical criteria
  • Provide clear criteria: Each response option should have distinct, unambiguous descriptions
  • Include remediation guidance: Help clients understand how to improve in areas where they score poorly

Category Organization

  • Group related topics: Questions within a category should evaluate related aspects of readiness
  • Balance coverage: Ensure categories cover different dimensions of the assessment topic
  • Consider weight distribution: Decide whether all categories should have equal importance

Scoring Instructions

  • Be objective: Provide clear criteria for how to evaluate each question
  • Consider context: Account for different client sizes and industries
  • Focus on outcomes: Emphasize business results rather than just technical implementation

üîß Advanced Features

Weight Distribution

The assessment system supports flexible weighting through the is_category_weight_evenly_distributed and is_question_weight_evenly_distributed fields. When set to true, all categories or questions within a category will have equal weight in the final score calculation.

Multi-Response Criteria

The criterion_label_type_enum field supports different response types. Currently, "MultiResponse" is the standard type that allows clients to select from the four standard response levels.

Remediation Integration

The remediation_tips field provides actionable guidance that can be integrated into recommendation engines and action planning tools.

üöÄ Getting Started

To create your first assessment template:

  1. Define your assessment scope: What area of client readiness are you evaluating?
  2. Identify key categories: Break down the assessment into logical groupings
  3. Create questions: Develop specific, measurable questions for each category
  4. Define response criteria: Ensure each question has clear, distinct response options
  5. Add guidance: Include remediation tips and scoring instructions
  6. Test and refine: Validate the assessment with sample clients

üí° Pro Tip

Start with a simple assessment structure and gradually add complexity. It's easier to expand an assessment than to simplify an overly complex one.
