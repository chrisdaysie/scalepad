‚è∞ Timestamp Management

The lastUpdated field controls when the report appears to have been last updated:

"lastUpdated": 1755105701

Timestamp Guidelines:

  • Format: Unix timestamp in seconds (not milliseconds)
  • Current Time: Use Math.floor(Date.now() / 1000) for current timestamp
  • Past Dates: Subtract seconds for past dates (86400 = 1 day)
  • Realistic Spread: Vary timestamps across reports to show recent activity

Example Timestamps:

  • 1 day ago: 1755105701
  • 2 days ago: 1755019301
  • 3 days ago: 1754932901

üéØ Best Practices

Content Guidelines

  • Executive Summary: 2-3 sentences describing overall performance and key achievements
  • Categories: 3-4 key areas with realistic scores (0-100)
  • Risks: 3-4 specific, actionable risks with clear business impact
  • Insights: 3-4 data-driven observations with metrics and trends
  • Recommendations: 3-4 actionable improvement suggestions with priority and effort

Naming Conventions

  • Report IDs: Use qbr-report-[platform-name] format
  • Company Names: Use uppercase for display
  • File Names: Use kebab-case (e.g., qbr-report-backup-radar.json)

Data Quality

  • Scores: Use realistic percentages (0-100) based on actual performance
  • Metrics: Include specific numbers and percentages where possible
  • Currency: Use consistent formatting ($X,XXX.XX)
  • Status Values: Use consistent terminology across all items

Report Types

  • Individual Reports: Focus on single platform or service (e.g., backup monitoring)
  • Aggregate Reports: Comprehensive multi-platform analysis with additional sections

‚ùå Common Mistakes to Avoid

Critical Validation Errors

  • Missing Required Fields: All required fields must be present
  • Invalid Status Values: Use only "satisfactory", "needs-attention", or "at-risk"
  • Invalid Severity Levels: Use only "high", "medium", or "low"
  • Invalid Priority Levels: Use only "critical", "high", "medium", or "low"
  • Invalid Trend Values: Use only "positive", "negative", or "neutral"
  • Score Range: Category scores must be between 0-100
  • String Values: All text must be properly quoted strings
  • Timestamp Format: Use seconds, not milliseconds for lastUpdated
