üèóÔ∏è JSON Structure

Root Object

The JSON file contains a single root object with the QBR report data:

{ "id": "qbr-report-example", "title": "Example Report", ... // other fields }

Required Fields

These fields are required for all QBR reports:

Field Type Required Description
id string Required Unique identifier for the report (e.g., "qbr-report-backup-radar")
title string Required Display title of the QBR report
company string Required Company name in uppercase (e.g., "EXAMPLE COMPANY")
dateRange string Required Report period and type (e.g., "Q2 2025 ‚Ä¢ Business Intelligence")
monthlyInvestment string Required Monthly investment amount (e.g., "$2,500")
managedEndpoints string Required Number of managed endpoints (e.g., "25 Managed Endpoints")
type string Required Report type: "individual" or "aggregate"
description string Required Brief description of the report purpose
executiveSummary string Required 2-3 sentence executive summary of performance
categories array Required Array of performance categories
enhancedSections object Required Object containing risks, insights, and recommendations

üìÇ Performance Categories

Categories group related performance metrics together. Each category has a score and multiple items.

Field Type Required Description
name string Required Category name (e.g., "Security Posture", "Backup Success")
score number Required Category score (0-100)
items array Required Array of category items

Example Category

{ "name": "Security Posture", "score": 85, "items": [ { "name": "Endpoint Protection", "status": "satisfactory" }, { "name": "Email Security", "status": "satisfactory" } ] }

üìù Category Items

Individual items within each category that represent specific metrics or assessments.

Field Type Required Description

status string Required Status: "satisfactory", "needs-attention", or "at-risk"

[ { "name": "Endpoint Protection", "status": "satisfactory" }, { "name": "Email Security", "status": "satisfactory" }, { "name": "Access Controls", "status": "needs-attention" } ]

Field Type Required Description

Insights

Field Type Required Description
title string Required Insight title

trend string Required Trend: "positive", "negative", or "neutral"

Recommendations

Field Type Required Description
title string Required Recommendation title
description string Required Detailed recommendation description
priority string Required Priority: "critical", "high", "medium", or "low"
effort string Required Implementation effort: "High", "Medium", or "Low"
impact string Required Expected business impact

üéØ Best Practices

Content Guidelines

  • Executive Summary: 2-3 sentences describing overall performance and key achievements
  • Categories: 3-4 key areas with realistic scores (0-100)
  • Risks: 3-4 specific, actionable risks with clear business impact
  • Insights: 3-4 data-driven observations with metrics and trends
  • Recommendations: 3-4 actionable improvement suggestions with priority and effort

Naming Conventions

  • Report IDs: Use qbr-report-[platform-name] format
  • Company Names: Use uppercase for display
  • File Names: Use kebab-case (e.g., qbr-report-backup-radar.json)

Data Quality

  • Scores: Use realistic percentages (0-100) based on actual performance
  • Metrics: Include specific numbers and percentages where possible
  • Currency: Use consistent formatting ($X,XXX.XX)
  • Status Values: Use consistent terminology across all items

Report Types

  • Individual Reports: Focus on single platform or service (e.g., backup monitoring)
  • Aggregate Reports: Comprehensive multi-platform analysis with additional sections

‚ùå Common Mistakes to Avoid

Critical Validation Errors

  • Missing Required Fields: All required fields must be present
  • Invalid Status Values: Use only "satisfactory", "needs-attention", or "at-risk"
  • Invalid Severity Levels: Use only "high", "medium", or "low"
  • Invalid Priority Levels: Use only "critical", "high", "medium", or "low"
  • Invalid Trend Values: Use only "positive", "negative", or "neutral"
  • Score Range: Category scores must be between 0-100
  • String Values: All text must be properly quoted strings
